# Img2Img

一款完全在浏览器本地运行的高质量图片格式转换工具，零上传、零依赖后端，所有处理均在客户端完成。

## 功能特性

### 格式转换
| 输出格式 | 说明 | 特性 |
|---------|------|------|
| **PNG** | 无损压缩 | 支持透明通道，适合截图、图标 |
| **JPEG** | 有损压缩 | 可调质量(1%~100%)，适合照片 |
| **WebP** | 高效压缩 | 可调质量，同质量下体积更小 |
| **BMP** | 无压缩位图 | 原始像素数据，文件较大 |
| **ICO** | Windows 图标 | 自动生成 16/32/48px 多尺寸 |
| **GIF** | 图形交换格式 | 输出单帧静态图 |

### 输入支持
PNG、JPEG、WebP、BMP、GIF、SVG、ICO 等所有浏览器可识别的图片格式。

### 其他能力
- **质量调节** — JPEG / WebP 格式支持 1%~100% 压缩质量滑块
- **尺寸调整** — 自定义输出宽高，支持锁定/解锁宽高比
- **批量操作** — 多选上传，文件列表独立切换预览与移除
- **大小对比** — 转换后实时展示原始大小 vs 转换后大小及变化百分比
- **实时预览** — 原图/转换结果 Tab 切换，棋盘格背景查看透明度
- **图片信息** — 显示格式、尺寸、文件大小、色深等元数据

## 使用方式

### 方式一：直接打开
双击 `index.html` 文件，在任何现代浏览器中即可使用。

### 方式二：本地服务器（推荐）
```bash
# 使用 Python
python -m http.server 8080

# 使用 Node.js
npx serve .

# 使用 PHP
php -S localhost:8080
```
然后访问 `http://localhost:8080`。

> 为什么推荐本地服务器？部分浏览器对 `file://` 协议下的 Canvas `toBlob` 有限制，通过 HTTP 服务可避免此问题。

## 技术实现

### 核心技术栈
- **HTML5 Canvas API** — 图片绘制与格式导出
- **FileReader API** — 本地文件读取
- **Blob / URL.createObjectURL** — 文件生成与预览
- **Drag & Drop API** — 拖拽上传
- **DataView / ArrayBuffer** — ICO 二进制文件手工拼装

### ICO 生成原理
ICO 文件不是浏览器原生支持的导出格式，工具通过以下方式实现：

1. 将原图缩放为 16×16、32×32、48×48 三个尺寸
2. 分别导出为 PNG 格式的 `ArrayBuffer`
3. 按照 ICO 文件规范（6 字节头 + 每图 16 字节目录项 + PNG 数据）手工拼装二进制
4. 最终输出为 `image/x-icon` 类型的 Blob

### JPEG 透明处理
JPEG 格式不支持 Alpha 通道。转换时自动检测并在绘制前填充白色背景，避免透明区域变黑。

### 浏览器兼容性
| 浏览器 | 最低版本 | 备注 |
|--------|---------|------|
| Chrome | 51+ | 完整支持 |
| Firefox | 96+ | WebP 导出需 96+ |
| Safari | 16.4+ | WebP 导出需 16.4+ |
| Edge | 79+ | 完整支持 |
